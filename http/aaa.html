Summary:

This plan is created for libvirt features, especially for new features in RHEL6.2. It base on "RHEL6.2 Libvirt IEEE Test Plan".
<https://riddler.bne.redhat.com/IEEE_Test_Plans/en-US/RHEL/6.1/html/Libvirt_IEEE_Test_Plan/index.html>

You could refer to "Chapter 10. Test design" for details.

 

1.Hot to use the scirpt "virt-pki-admin.pl" in attachements to set up   tls/x509 envirments.

1.1 Perpare two machine
Here i use
10.66.0.132 dhcp-0-132.pek.redhat.com as the client.And the script is in this machine.
10.66.82.249 intel-q9400-4-1.englab.nay.redhat.com as the server

1.2 create the ca's file
first change the script's mode
in 10.66.0.132
#chmod 777 virt-pki-admin.pl
#virt-pki-admin.pl init "my cn"

1.3 create the server's and client's file
#virt-pki-admin add_server "test server" intel-q9400-4-1.englab.nay.redhat.com
#virt-pki-admin add_client "test client" dhcp-0-132.pek.redhat.com

1.4 send the files
#virt-pki-admin deploy_server intel-q9400-4-1.englab.nay.redhat.com
#virt-pki-admin deploy_client dhcp-0-132.pek.redhat.com

Note:If you send the file to local , need to add ip and hostname in /etc/hosts.

1.5 config the libvirt
in server
1. Turn on --listen flag in /etc/sysconfig/libvirtd on destination host
2. config /etc/libvirt/libvirtd.conf
listen_tls = 1
auth_tls="none"
3. service libvirtd restart
both in server and client
#iptables -F

1.6
Till now ,
#virsh -c qemu+tls://intel-q9400-4-1.englab.nay.redhat.com/system
can work.
